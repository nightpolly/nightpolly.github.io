This post is about: My Sublime Text 2 setup—settings and installed packages.

Nov. 1, 2013 · frontendings · last updated: Dec. 28, 2013

How I Do Sublime Text 2

Sublime Text is an extremely customizable text editor, and it’s my code editor of choice. All of its settings are stored in JSON (sort of) files, and you can enhance it even further with external packages.

Sublime Text Packages

They’re plugins you can install to suit your workflow, the languages you work with, your UI preferences. You can practically make Sublime Text your own personal code editor.

Essential

Package Control is the easiest way to find and manage packages. The website got a fresh new design recently and, if you’re anything like me, you’d love to spend some time browsing around. There is always cool stuff there.

SideBarEnhancements gives you the context menu that’s missing from Sublime’s sidebar. It has the basic options you’d expect, but also allows you to do things that normally can’t be done inside the editor: move a file to a different folder, rename it, copy its name or path, etc.

AdvancedNewFile allows you to create files right where you need them. Bring up the input field with a key binding, and type the file path and name. If you specify a directory that doesn’t exist, it will be created for you.

Lookin’ good

Sublime Text has some nice color schemes for you to choose from, but you can also install more, or even write your own.

Dayle Rees Color Schemes is a very popular collection of color schemes, and base16 is worth a try, too. I also use Focus, which is useful when you want to only concentrate on the content.

Dec. 28, 2013

Spacegray is a beautiful UI theme for Sublime Text. Search for “Theme - Spacegray” in Package Control to try it.

Power-coding

Emmet is definitely worth checking out if you work with HTML and CSS. You can do many things with it, here’s just one quick example:

HTML

<!-- input -->
section.main>(h1>a)+ul.contents>li*4>a{Part $}

<!-- output -->
<section class="main">
  <h1><a href=""></a></h1>
  <ul class="contents">
    <li><a href="">Part 1</a></li>
    <li><a href="">Part 2</a></li>
    <li><a href="">Part 3</a></li>
    <li><a href="">Part 4</a></li>
  </ul>
</section>

Emmet LiveStyle allows you to edit CSS in Chrome DevTools (you’ll need also the extension for Chrome), and have the changes show up in your file in Sublime Text. It’s kind of magical, this one.

Inc-Dec-Value: use keyboard shortcuts to increase or decrease numbers, hex-codes, dates, change value to the opposite (true ↔ false), cycle through days of the week, month names, etc., turn a string into upper, lower, or capital case.

Additional syntaxes

Sublime doesn’t have built-in support for all of the syntaxes out there. But if the one you want is missing, there’s a big chance someone has made a plugin for it.

Sass for TextMate / Sublime Text 2 is the one I use for Sass and SCSS syntax highlighting. A similar package you can check out is Syntax Highlighting for Sass.

Siteleaf Liquid Syntax provides support for Liquid, as well as some useful snippets.

Other

Gist is a plugin for creating, well, gists—just select some code and hit the keyboard shortcut. You can also open and insert gists you already have on GitHub.

Sublime Text Caniuse is helpful if you need a reminder of which browser supports what CSS property.

Nov. 12, 2013

FindKeyConflicts can help you locate keyboard shortcuts mapped to more than one package.

Settings

You can access your settings file from Preferences → Settings - User.

The defaults are a handy reference, with comments and everything: Preferences → Settings - Default. You can’t put comments in the user settings file though (well you can, but they’ll be lost when you close it).

So here’s what my settings look like (most are self-explanatory and / or described in the default settings file; some depend on external packages and other stuff):

JSON

{
  "auto_complete_commit_on_tab": true,
  "auto_complete_selector": "source, text",
  "bold_folder_labels": true,
  "caret_style": "wide",
  "color_scheme": "Packages/awesome/LastNight.tmTheme",
  "default_line_ending": "unix",
  "detect_slow_plugins": false,
  "ensure_newline_at_eof_on_save": true,
  "file_exclude_patterns":
  [
    ".DS_Store"
  ],
  "folder_exclude_patterns":
  [
    ".git"
  ],
  "font_face": "Monaco",
  "font_options":
  [
    "directwrite",
    "no_bold",
    "no_italic"
  ],
  "font_size": 11,
  "highlight_line": true,
  "ignored_packages":
  [
    "Vintage"
  ],
  "line_padding_bottom": 1,
  "line_padding_top": 1,
  "overlay_scroll_bars": "enabled",
  "tab_size": 2,
  "theme": "Spacegray.sublime-theme",
  "translate_tabs_to_spaces": true,
  "trim_trailing_white_space_on_save": true,
  "word_wrap": true
}

I’ve also backed this up on GitHub.

A few notes:

  • "color_scheme": "Packages/awesome/LastNight.tmTheme"
    There’s no such directory in Packages—I just like to change my color scheme from time to time, so I’ve copied all my favorites into a single folder (awesome) for quick access. LastNight is from Dayle Rees Color Schemes.
  • directwrite in font_options is Windows-only. Enabling it improves the text rendering.
